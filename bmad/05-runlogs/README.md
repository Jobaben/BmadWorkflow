# BMAD Session Runlogs

This directory contains daily session logs for all BMAD role activations.

## File Naming

```
session-YYYY-MM-DD.md
```

Example: `session-2024-01-15.md`

## Log Entry Format

Each session creates an entry with:
- Timestamp
- Role activated
- Input artifacts read
- Actions taken
- Outputs created
- Session status

## Purpose

- **Audit Trail**: Track all workflow activities
- **Context Continuity**: Understand prior decisions
- **Debugging**: Trace issues to specific sessions
- **Progress Tracking**: See daily activity

## Usage

Runlogs are automatically created/updated by each role session via the `skills/runlog.md` skill.

---

_Do not manually edit runlog files._
